# GalgameCharacterSkills

本地运行的长文本角色人设蒸馏工具。

给定一段文本（可以是 Galgame 剧情脚本、小说或其他），自动将其中指定角色蒸馏为 skills 或适用于 SillyTavern 的角色卡。

## 特点

1. 基于给定文本 **自动蒸馏** 选定角色人设
2. 支持使用 **VNDB** 相关信息进行信息增强
3. 支持 **长文本**，自动进行 **切片与合并**
4. 支持 **并行** 切片蒸馏与 **保存、恢复** 蒸馏记录
5. 使用兼容 **OpenAI** 接口格式的 API 进行蒸馏
6. 允许在分析文本过程中添加 **自定义指令**

## 快速开始

把本项目拉取到本地并进入项目根目录后，按以下步骤操作

### 环境要求

- Python 3.10 及以上（推荐 3.10）
- Bun 1.2 及以上

建议先进入 Python 虚拟环境（如 `conda` 或 `venv`）以避免安装依赖污染全局环境。

### 安装依赖

```
bun run setup
```

该命令会在当前激活的 Python 环境中执行：

```
pip install -e .
```

如果当前未检测到虚拟环境，脚本会要求你二次确认，以避免误装到系统 Python

### 运行测试

运行全部测试：

```
bun run test
```

只运行特定模块 `xxx` 的测试：

```
bun run test:xxx
```

### 查看覆盖率

查看整个项目覆盖率：

```
bun run cov
```

只运行特定模块 `xxx` 的测试：

```
bun run cov:xxx
```

## 常见问题

### 不想安装 bun.js，可以正常使用 / 测试吗？

目前 bun.js 只起到封装命令的效果，因此可以阅读 `package.json` 内的等效指令执行。

### 蒸馏速度很慢？

蒸馏速度取决于 **调用模型的响应速度** 与 **所处网络**，可以在蒸馏前 **去除无关数据** 以减少模型分析负担。

未来将会在页面内添加请求锁，在目前阶段 **反复点击按钮可能导致额外 Token 开销**，敬请注意。

### 模型请求失败？

排查指南：

1. 确认配置参数正确，包括 BaseURL 是否包含 `v1/` 后缀，API Key 是否可用等
2. 确认模型可正常调用，可通过 **自行检查** 或 **点击 WebUI 页面内的连通性测试按钮** 进行检查。（注意：连通性检测本质上是 **发送一段文本给模型并获取回复**，可能有 **极少量额外 Token 开销**）
3. 确认传入数据符合模型的调用规则。可能因为文本中 **存在敏感信息** 被审查拦截导致请求失败。
4. 如果上述排查步骤没有解决你的问题，请提供完整的 **运行时设备信息** 与 **运行日志**（在 `output/logs/` 目录下），并在 Github 页面 **提出 Issues** 进行反馈。
