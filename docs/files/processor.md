# FileProcessor

## 1. 核心文件

- `files/processor.py`


## 2. 职责

`FileProcessor` 负责：

- 扫描 `resource/` 目录
- 保存上传文件
- 计算 token 数
- 估算切片数
- 合并并切分多个文本文件


## 3. 设计特点

它当前是一个“面向本地工具”的综合型文件处理器。

优点是：

- 使用简单
- 对当前项目足够实用

代价是：

- token 计算与切片策略都集中在一个类中
- 后续若策略更复杂，可能需要进一步拆分


## 4. 重要约定

- 支持的文件类型主要是 `.txt` / `.md`
- 资源目录默认为项目根下的 `resource/`
- 分词器当前使用 `tiktoken` 的 `cl100k_base`
